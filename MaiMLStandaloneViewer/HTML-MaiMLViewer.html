                            1. 概要と基本機能

                        
MaiML Viewer は、計測分析データ形式 "MaiML"(Measurement Analysis Instrument Markup
                            Language) を可視化・分析するためのWebアプリケーションです。

                        
    
                            
  • グラフ可視化: 実験プロセス（Petri Net）とデータの関係をノードグラフで表示します。
    • 
                            
  • メタデータ抽出: ドキュメント情報、テンプレート定義、計測結果を一元管理・表示します。
    • 
                            
  • データプレビュー: 埋め込みファイル（画像、CSV、テキスト、TIFF等）をブラウザ上でプレビューします。
    • 
                            
  • AI要約 (Gemini): Google Gemini API を利用して、実験データの要約やレポートを生成します。
    • 
                        

                    


                    

                        

                            2. 基本操作

                        
📂 ファイル読み込み
                            (Upload)

                        
画面左上の 📂UPLOAD…
                            ボタン、または画面へのドラッグ＆ドロップでファイルを読み込みます。

                        
    
                            
  • 対応形式: .maiml およびこれらを含む .zip ファイル。
    • 
                            
  • ZIP対応: ZIPファイルをアップロードすると、内部のMaiMLファイルと画像/データファイルを自動的に紐付けて展開します。
    • 
                        


                        
🖱️ グラフ操作

                        
    
                            
  • 移動 (Pan): 何もないところをドラッグ。
    • 
                            
  • 拡大縮小 (Zoom): マウスホイール。
    • 
                            
  • ノード移動: 各ノード（丸や四角）をドラッグして配置を調整できます。
    Drag
                                    Edit チェックボックスで有効/無効を切り替えます。
    • 
                            
  • レイアウト自動調整: Auto
                                    Layout (力学モデル) または Dagre
                                    Layout (階層型) で配置を整理できます。
    • 
                        

                    


                    

                        

                            3. 画面の見方

                        
左パネル: Graph Stage
                        

                        
実験のワークフローとデータの構造を可視化します。

                        
    
                            
  • ⚪Place (円): 試料やデータの「状態」を表します。
                            
    • 
                            
  • ⬜Transition (長方形):
                                処理やイベントを表します。
    • 
                            
  • ---▶Template Ref
                                    (黄色の破線): データの「ひな形（テンプレート）」同士の参照関係です。
    • 
                            
  • ---▶Instance Ref
                                    (緑色の破線): 実際の「測定データ（インスタンス）」同士の参照関係です。
    • 
                            
  • ---Chain
                                    (橙色の破線): 異なるMaiMLファイル間の連鎖（chain）関係です。ハッシュ値による整合性検証も行います。
    • 
                            
  • ---Same UUID : Template
                                    (青紫の破線): 同一UUIDを持つTemplateノード同士の対応関係です。
    • 
                            
  • ---Same UUID : Instance
                                    (薄紫の破線): 同一UUIDを持つInstanceノード同士の対応関係です。
    • 
                        


                        
🔗 Link Visibility
                            ツールボックス

                        
グラフエリアの左上にある 🔗
                                Link Visibility ボタンをクリックするとツールボックスが展開されます。

                        
    
                            
  • 5種類の関係線（templateRef・instanceRef・chain・same uuid:Template・same
                                uuid:Instance）をチェックボックスで個別に表示/非表示切り替えできます。
    • 
                            
  • 関係線が重なって見づらい場合にOFFにすることで、必要な線だけを表示できます。
    • 
                            
  • チェックを外すと、対応する線とラベルが即座に非表示になります。
    • 
                            
  • ツールボックスを使わない場合は折り畳んでおけばグラフの表示を妨げません。
    • 
                        


                        
右上パネル: Metadata
                            Panel

                        
ドキュメント全体のメタデータ（作成者、UUID、説明など）を表示します。


                        
右下パネル: Details
                            Panel

                        
グラフ上で選択したノードの詳細情報を表示します。ノードをクリックすると、そこに関連付けられたテンプレートやインスタンスのリストが表示されます。

  • JSON (.json): 階層構造を維持した生データ形式。
    • 
                            
  • CSV (.csv): 他のツールでの解析用。

数値配列データ（スペクトル等）が含まれる場合、詳細パネルのラジオボタンでX軸・Y軸を選択し、📈

    • 📊 Bar（棒グラフ）: ヒストグラムなど離散データの表示に適しています。

      • 
                                
    
                            
    • 
                            
  • ズーム: グラフ上でマウスホイールを回転させると拡大・縮小できます。
    • 
                            
  • パン（移動）: グラフをドラッグすると表示範囲を移動できます。
    • 
                            
  • ズームリセット: グラフをダブルクリック、または 🔍
                                    Reset Zoom
                                ボタンで初期表示に戻ります。

                            🔐 5. XML復号機能 (JIS K0200)


                        
概要

                        
MaiML Studio で暗号化して出力した .maiml
                            ファイルを復号して表示できます。AES-256-GCM + PBKDF2-SHA256（W3C XML Encryption 1.1 / JIS
                            K0200 準拠）のパスワード方式のみ対応しています。

                        

                            ⚠️ パスワード方式（PBKDF2）のみ対応。 RSA や鍵ラップ方式で暗号化された要素は復号できません（⚠️ Unsupported
                            と表示されます）。
                        


                        
📢 暗号済みファイルの読み込み
                        

                        
暗号化要素を含む MaiML ファイルを読み込むと、ツールバー直下に通知バナーが表示されます。

                        
    
                            
  • 🔐 N encrypted element(s) found. — N 個の暗号化要素が検出されたことを示します。
    • 
                            
  • バナー右端の 🔓
                                    Decrypt All ボタンで全要素を一括復号できます。
    • 
                        


                        
🔓 個別復号の手順

                        
    
                            
  1. ノードをクリックして詳細パネルを表示します。
    2. 
                            
  2. 暗号化された行の Value 列に 🔓
                                    Decrypt ボタンが表示されます（tooltip: Decrypt (Password-based / PBKDF2 only)）。
                            
    3. 
                            
  3. ボタンをクリックするとパスワード入力モーダルが表示されます（初回のみ）。
    4. 
                            
  4. 正しいパスワードを入力すると復号され、Description と Value が表示されます。復号済みの行には
                                🔓 バッジが残ります。
    5. 
                            
  5. 2回目以降は同一セッション内でパスワードが自動使用されます。
    6. 
                        


                        
⚡ Decrypt
                            All（一括復号）

                        
    
                            
  • ノードクリック前でも使用可能： ファイル読み込み直後にバナーの Decrypt All
                                を押すとパスワードが事前保存されます。その後ノードをクリックすると暗号化行が自動復号されます。
    • 
                            
  • ノードクリック後： 現在表示中の全暗号化行をまとめて復号します。
    • 
                        


                        
🔑 パスワードキャッシュ

                        
    
                            
  • 一度入力したパスワードはセッション中メモリに保持され、次回から自動使用されます。
    • 
                            
  • localStorage 等には保存されず、ページ再読み込みで消去されます。
    • 
                            
  • 誤ったパスワードを入力するとキャッシュがクリアされ、再入力を求められます。
    • 
                        


                        
📊 復号後の表示とエクスポート
                        

                        
    
                            
  • ネスト構造（property / content / uncertainty の子要素）は復号後に自動展開され、テーブルに行として追加されます。
    • 
                            
  • 復号済みのデータは AI Summary および Export Metadata の対象に含まれます。
                            
    • 
                            
  • 数値配列 content 要素が展開された場合、PLOT 機能でグラフ表示も可能です。
    • 
                        

                    


                    

                        

                            🔏 6. XML署名検証機能 (ds:Signature)


                        
概要

                        
W3C XML Signature 準拠（ECDSA P-256）で署名された MaiML
                            ファイルを読み込んだ際、改ざんが行われていないか自動的に検証します。


                        
✅ 検証と結果表示

                        
検証結果は右上パネル（Document Metadata）の表の最上部に自動表示されます。

                        
    
                            
  • ✅
                                    Valid — 署名が有効かつ、データが改ざんされていません。
    • 
                            
  • ❌
                                    Invalid — ハッシュの不一致、または署名値の不一致が検出されました。ファイルの内容が改変されている可能性があります。
    • 
                            
  • ⚠️
                                    Unsupported — RSA 暗号など、現在のビューアが未対応のアルゴリズムの署名です（現状は ECDSA P-256 のみに対応）。
    • 
                            
  • 🔏
                                    Verifying... — ファイル読み込み中に表示される、検証処理中の状態です。
    • 
                        

                    


                


                

                    

                        

                            1. Overview & Basic Functions

                        
MaiML Viewer is a web application for visualizing and analyzing "MaiML"
                            (Measurement Analysis Instrument Markup Language) data files.

                        
    
                            
  • Visualization: Displays the experimental process (Petri Net) and data
                                relationships as a node graph.
    • 
                            
  • Metadata Extraction: Manages and displays document info, template
                                definitions, and measurement results.
    • 
                            
  • Data Preview: Previews embedded files (Images, CSV, Text, TIFF, etc.)
                                directly in the browser.
    • 
                            
  • AI Summary (Gemini): Uses Google Gemini API to summarize experimental
                                data and generate reports.
    •

                            🔐 5. XML Decryption (JIS K0200)


                        
Overview

                        
Decrypts and displays .maiml files encrypted by
                            MaiML Studio. Only AES-256-GCM + PBKDF2-SHA256 password-based decryption is
                            supported (W3C XML Encryption 1.1 / JIS K0200).

                        

                            ⚠️ Password-based (PBKDF2) decryption only. Elements encrypted with RSA or
                            key-wrap methods cannot be decrypted (shown as ⚠️ Unsupported).
                        


                        
📢 Loading an
                            Encrypted File

                        
When loading a MaiML file containing encrypted elements, a notification banner appears just
                            below the toolbar.

                        
    
                            
  • 🔐 N encrypted element(s) found. — Indicates N encrypted elements were
                                detected.
    • 
                            
  • Click the 🔓
                                    Decrypt All button on the right side of the banner to decrypt all elements at
                                once.
    • 
                        


                        
🔓 Individual
                            Decryption

                        
    
                            
  1. Click a node to open the Details Panel.
    2. 
                            
  2. Encrypted rows show a 🔓
                                    Decrypt button in the Value column (tooltip: Decrypt (Password-based /
                                    PBKDF2 only)).
    3. 
                            
  3. Clicking the button shows a password input modal (first
                                time only).
    4. 
                            
  4. On success, the description and value are revealed. A 🔓
                                badge remains on the row to indicate it was previously encrypted.
    5. 
                            
  5. The same password is reused automatically for subsequent decryptions within the same
                                session.
    6. 
                        


                        
⚡ Decrypt All

                        
    
                            
  • Before clicking a node: Pressing Decrypt All saves the password in
                                advance. Encrypted rows are automatically decrypted as you navigate to each node.
    • 
                            
  • After clicking a node: All currently displayed encrypted rows are
                                decrypted at once.
    • 
                        


                        
🔑 Password Cache
                        

                        
    
                            
  • The password entered is cached in memory for the session and reused automatically.
    • 
                            
  • It is never saved to localStorage or any persistent storage, and is cleared on page
                                reload.
    • 
                            
  • An incorrect password clears the cache and prompts re-entry.
    • 
                        


                        
📊 Display and
                            Export After Decryption

                        
    
                            
  • Nested structures (child property / content / uncertainty elements) are automatically
                                expanded and added as rows to the table.
    • 
                            
  • Decrypted data is included in AI Summary and Export
                                    Metadata.
    • 
                            
  • If numeric array content elements are revealed, the PLOT feature
                                becomes available for those rows.
    • 
                        

                    


                    

                        

                            🔏 6. XML Signature Verification (ds:Signature)


                        
Overview

                        
Automatically verifies W3C XML Signature compliant (ECDSA
                            P-256) signatures when a signed MaiML file is loaded to detect any tampering.


                        
✅ Verification and
                            Display

                        
The verification result is automatically displayed at the very top of the table in the
                            top-right panel (Document Metadata).

                        
    
                            
  • ✅
                                    Valid — The signature is valid and the data has not been tampered with.
    • 
                            
  • ❌
                                    Invalid — Hash mismatch or signature value mismatch detected. The file
                                content may have been altered.
    • 
                            
  • ⚠️
                                    Unsupported — The signature uses an unsupported algorithm, such as RSA.
                                Currently, only ECDSA P-256 is supported.
    • 
                            
  • 🔏
                                    Verifying... — Displayed immediately after loading while the verification
                                process is running.
    •
